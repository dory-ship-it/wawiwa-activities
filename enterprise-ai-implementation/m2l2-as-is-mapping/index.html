How to build it

Rows are roles, columns are time, activities live in role rows, arrows show handoffs, diamonds mark decisions. Class example: 3 rows (Requester / Manager / Finance), 4 columns (Submit / Review / Approve / Pay).

Where the example is too tidy: a real NorthRiver process has at least one looped arrow (re-approval), at least one cross-row escalation, and at least one activity nobody owns. Map that version.
